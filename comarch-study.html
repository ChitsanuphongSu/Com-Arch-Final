Q1 — Basic Computer: 32-bit Word
Ch5Instruction FormatRegisters

โจทย์กำหนดว่า มี 32 bits ต่อ 1 คำ (word) แทนที่จะเป็น 16-bit ของ Basic Computer เดิม แล้วถามคุณสมบัติต่าง ๆ

พื้นฐานที่ต้องเข้าใจก่อน

Basic Computer เดิม (16-bit word) มี memory 4096 words = 212 → ต้องใช้ 12 bits เป็น address

Format ของ instruction ปกติ:

1.1) OP-Code กี่ bit?

ถ้า word = 32 bit และยังคงใช้ 1 bit สำหรับ I (indirect mode) และ address field ต้องรองรับ memory ขนาดเดิม (หรือตามโจทย์กำหนด):

วิธีคิด:
ถ้า memory = 4096 = 212 → Address = 12 bit, I = 1 bit
ดังนั้น OP-code = 32 − 1 (I) − 12 (Address) = 19 bit

แต่ถ้าโจทย์ระบุ memory ใหม่ เช่น memory = 220:
OP-code = 32 − 1 − 20 = 11 bit
✓ ตอบ: นับจาก 32 − 1 (I) − จำนวน bit ของ Address = จำนวน bit OP-code
เช็คโจทย์ให้ดีว่า memory กี่ words → ใช้กี่ bit เป็น address
1.2) Register กี่ bit?

Register ใน Basic Computer มีขนาดตาม function:

Registerขนาด (เดิม 16-bit)ขนาด (32-bit word)
AC, DR, IR, TR16 bit32 bit (= word size)
AR, PC12 bit (= address bits)ขึ้นอยู่กับ address field ใหม่
INPR, OUTR8 bit (character)8 bit (ไม่เปลี่ยน)
✓ ตอบ: AR และ PC = จำนวน bit ของ address field | AC, DR, IR, TR = 32 bit
1.3) เกี่ยวกับ bit — Addressing Mode

ใน instruction format ของ Basic Computer:

  • I = 0 → Direct Addressing: EA = address field ใน instruction
  • I = 1 → Indirect Addressing: EA = M[address field]
วาดให้ดู:
Direct: Instruction[addr] → M[addr] = Operand
Indirect: Instruction[addr] → M[addr] = addr2 → M[addr2] = Operand
✓ bit 31 (MSB ถ้า 32-bit) = I (indirect flag), 1 bit เสมอ
1.4) จำนวน Register / Register ที่มี

Basic Computer มี 8 registers หลัก:

AR (12 bit) Address Register
PC (12 bit) Program Counter
DR (16 bit) Data Register
AC (16 bit) Accumulator
IR (16 bit) Instruction Register
TR (16 bit) Temporary Register
INPR (8 bit) Input Register
OUTR (8 bit) Output Register
1.5) จำนวน Instruction ที่เป็นไปได้

จำนวน instruction = 2จำนวน bit OP-code

ตัวอย่าง: ถ้า OP-code = 19 bit → จำนวน instruction = 219 = 524,288 instructions
แต่ใน Basic Computer เดิม: OP-code 3 bit → 23 = 8 memory-reference instructions + register/IO instructions (OP=111)
✓ ตอบ: 2n โดย n = จำนวน bit OP-code

AR (Address Register) — หน้าที่และการทำงาน

AR = ตัวที่บอกว่า "จะไปหยิบข้อมูลจาก memory ตรงไหน" (holds the memory address)

  • AR เป็น 12-bit register (เพราะ address = 12 bit สำหรับ 4096 locations)
  • ทุกครั้งที่อ่าน/เขียน memory → ต้องใส่ address ลง AR ก่อน
  • เงื่อนไขการ Load AR: AR ← PC, AR ← IR(0-11)
ตัวอย่าง signal ที่ activate AR:
R₀T₄: AR ← 0, TR, PC (Load AR จาก PC)
D₅T₅: AR ← IR(0-11) (Load AR จาก instruction field)
สัญญาณที่ทำให้ AR ทำงาน = R₀ AND T₄, หรือ D₅ AND T₅ → ผ่าน OR gate → LD ของ AR
✓ AR = ดูที่ตั้งไว้ว่าตรองไหน — เก็บ address ของ memory ที่จะเข้าถึง
วงจร Load AR (ตาม Signal)

หลักการ: หลาย AND gate → OR gate เดียว → LD (Load) ของ AR
ถ้าเงื่อนไขใดๆ เป็น 1 → AR จะถูก load

Q3 — Postfix / Prefix / 3-Address Instruction
ExpressionAddressing

แปลงนิพจน์ระหว่าง Infix ↔ Postfix ↔ Prefix และเขียน 3-address instruction

Prefix / Infix / Postfix — สรุปหลัก
ชนิดตำแหน่ง Operatorตัวอย่าง A+B
Prefixก่อน operand+ A B
Infixระหว่าง operandA + B
Postfixหลัง operandA B +
3.1) แปลง Postfix → Infix: A+B*C-D

โจทย์ (Postfix): A B C * + D -

วิธีทำ — ใช้ Stack
1

2
อ่าน B → push "B"
3
อ่าน C → push "C"
4

5

อ่าน D → push "D"
7
อ่าน - → pop (A+(B*C)), D → push "((A+(B*C))-D)"
✓ Infix: (A + (B*C)) - D
3.1) แปลง Postfix → Prefix (ตัวอย่างจากโจทย์)

โจทย์ตามในเอกสาร: A+B*C-D (Infix) → แปลงเป็น Prefix

วิธีจำเร็ว — Prefix:
เขียน operator ก่อน operand ซ้าย+ขวาเสมอ
A + (B*C) - D = - + A * B C D
✓ Prefix: - + A * B C D
✓ Postfix: A B C * + D -
3.2) เขียนเป็น 3-Address Instruction

สำหรับ expression: (A + B) * C - D / E

; 3-Address format: OP dst, src1, src2 MUL T1, A, B ; T1 = A + B (หมายเหตุ: ตามโจทย์) ADD T1, A, B ; T1 = A + B DIV T2, D, E ; T2 = D / E MUL T3, T1, C ; T3 = T1 * C SUB T4, T3, T2 ; T4 = T3 - T2 = result
3-Address = ระบุ destination และ 2 source ต่อ instruction
ข้อดี: ไม่ต้องใช้ตัวแปรชั่วคราวเยอะ, คำสั่งน้อย
ข้อเสีย: instruction ยาวมาก (ต้องเก็บ 3 addresses)

Q5 — Pipeline: Speedup & Comparison
Ch9PipelineSpeedup

คำนวณ Speedup ของ Pipelined vs Non-Pipelined, วาด pipeline diagram, เปรียบเทียบ

Pipeline — สูตรสำคัญ
τ_k = (k + n - 1) × t_p
Symbolความหมาย
nจำนวน task ที่ต้องทำ
kจำนวน stage ของ pipeline
t_pเวลาต่อ 1 clock cycle ของ pipeline
τ_kเวลารวมที่ pipeline ใช้
t_nเวลา 1 instruction แบบ non-pipeline
τ_1 = n × t_nเวลารวมแบบ non-pipeline
S_k = (n × t_n) / ((k + n - 1) × t_p)
กรณีพิเศษ: ถ้า t_n = k × t_p (แต่ละ stage เท่ากัน) และ n → ∞
lim S_k = t_n / t_p = k
→ Speedup สูงสุดที่ได้คือ k เท่า

5.2) วิธีเพิ่ม Speedup

  • ลด t_p ให้น้อยลง (clock เร็วขึ้น)
  • ทำให้แต่ละ stage ใช้เวลาเท่ากัน (balanced pipeline)
  • หลีกเลี่ยง pipeline hazards (stall)
5.3) เปรียบเทียบ 2 Pipeline

ถ้า n → ∞: Speedup ≈ t_n/t_p
Pipeline B มี t_p น้อยกว่า → Speedup มากกว่า แม้จะมี stages เพิ่ม
แต่ต้องระวัง overhead ของ pipeline registers ที่เพิ่มขึ้น
S_A/S_B = t_p(B) / t_p(A) = 0.1/0.2 = 0.5 (A ช้ากว่า B)
Q6 — Pipelined D[i] = A[i]×B[i] + A[i]×C[i]
Ch9Pipeline DesignSpeedup

โจทย์จะให้ทำ D[i] = A[i]*B[i] + A[i]*C[i] สำหรับ i=1,2,...,n โดยใช้ pipeline

วิเคราะห์ Operation

D[i] = A[i]×B[i] + A[i]×C[i]

แต่ละ iteration ต้องทำ:

  • Segment 1: Load A[i] และ B[i] จาก memory
  • Segment 2: Multiply → R3 = R1 × R2, Load C[i] → R4
  • Segment 3: Add → R5 = R3 + R4
; Pipeline Registers R1 ← A[i] ; Segment 1: Load A R2 ← B[i] ; Segment 1: Load B R3 ← R1 × R2 ; Segment 2: Multiply A×B R4 ← C[i] ; Segment 2: Load C R5 ← R3 + R4 ; Segment 3: Add ; D[i] = R5

สมมติ t_n = 3t_p (แต่ละ stage ใช้เวลา t_p เท่ากัน)

Q8 — Pipeline Hazards
Ch9HazardsStall

3 ประเภทของ Pipeline Hazard และวิธีแก้ไข

Major Hazards in Pipelined Execution
1. Structural Hazards

Resource Conflicts

เกิดเมื่อ hardware resource ถูกใช้พร้อมกันโดย 2 instructions ที่ต่างกัน

ตัวอย่าง: memory เดียวกันถูกใช้ทั้ง fetch instruction และ load data พร้อมกัน

แก้ไข:
• แยก Instruction Memory และ Data Memory
• Pipeline interlock (stall/bubble)
2. Data Hazards

Data Dependency Conflicts

Instruction ถัดไปต้องใช้ผลลัพธ์ของ instruction ก่อนที่ยังไม่เสร็จ

ตัวอย่าง:
ADD R1, R2, R3 ← ยังไม่เสร็จ
SUB R4, R1, R5 ← ต้องการ R1

แก้ไข:
• Data Forwarding / Bypassing
• NOP insertion (stall)
• Out-of-order execution
3. Control Hazards (Branch Hazards)

เกิดจาก Branch instruction ที่เปลี่ยน PC ทำให้ instruction ที่ fetch มาแล้วต้อง discard

Branch address dependency:
ไม่รู้ว่าจะ branch ไปไหนจนกว่าจะ execute instruction นั้น
→ flush pipeline
แก้ไข:
• Branch prediction (static/dynamic)
• Delayed branching
• Branch target buffer
